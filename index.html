Welcome!

This site is an in-progress project. The final aim is to display a gallery


flash_on

Functionality so Far

The website has a responsive layout in that it automatically can adjust to the browser's screen, making it adapt for mobile usage. Ths sidebar is useful for navigation. It remains visible for large-enough screens so that even when scrolling down on the page the user can easily navigate around. Files can be uploaded to the site (static, there is no backend yet), when it is an image it is displayed. The image can be zoomed in on by clicking it.

group

Interactivity

Currently the user can scroll through the page using the navigation links on the top header, or side navigation. Doing so will smoothly redirect them to the section on the page that the link refers to. The website currently takes in drag-and-drop files (of any kind), it checks if it is able to display a thumbnail for that file type (JPEG, PNG, GIF, ..). If it can, the image will be displayed after processing in a neat grid layout. The sidebar can be dragged in for mobile users. Furthermore, when clicking the 'about' link a modal wil open displaying information regarding the site, perhaps contact info at a later date.

Try it! drag an image onto the site, it will be displayed in Section 2. Alternatively you can click the upload button below

Playground

I am experimenting with different plugins and API's. I have settled on the Dropzone.js module for file handling. The website uses a Materialize template that uses a grid layout and jQuery plugins for interactive elements, that respond nicely for different screen sizes. I initially focused on having a robust deployment setup using Webpack for combining my files and modules into a single bundle, this includes loaders for files, HTML pages, Javascript, CSS and custom fonts. Using Babel I'm able to write my code in ECMAScript 6 and still support older browsers. This setup allows me to have a modularized approach while making it easy to compile the final site and upload it to github pages. Of course I use a git repository for version control, making it easy to revert when problems arise.
